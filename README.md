## 🎯 Aperçu

Le **Gestionnaire de Projets IA** est une application web moderne et complète conçue pour révolutionner la gestion de projets. Développée avec Streamlit, elle combine une interface utilisateur intuitive avec des fonctionnalités avancées de visualisation et d'analyse.

### 🎪 **Pourquoi cette application ?**

- ⚡ **Interface moderne** avec effets glassmorphism et animations fluides
- 📊 **Visualisations interactives** avec Plotly pour une analyse approfondie
- 🔄 **Vues multiples** : Kanban, Gantt, Calendrier, Tableau de bord
- 🤝 **CRM intégré** pour la gestion complète des relations clients
- 📦 **Gestion d'inventaire** avec système de mesures impériales/métriques
- 🎨 **Design responsive** adapté mobile et desktop
- 🚀 **Déploiement facile** sur cloud avec Render/Heroku

## ✨ Fonctionnalités

### 🏠 **Tableau de Bord Intelligent**

- 🎯 **KPI visuels** : Chiffre d'affaires, progression, performances
- 📊 **Graphiques interactifs** : Répartition par statut, priorité, client

- ✅ **CRUD complet** : Créer, lire, modifier, supprimer

## 🚀 Installation

### 📋 **Prérequis**
- Python 3.11+ 
- pip (gestionnaire de paquets Python)
- Git

### 💻 **Installation Locale**

1. **Cloner le repository**
   ```bash
   git clone https://github.com/Estimation79/gestion-projets-ia-streamlit.git
   cd gestion-projets-ia-streamlit
   ```

2. **Créer un environnement virtuel (recommandé)**
   ```bash
   python -m venv venv
   
   # Windows
   venv\Scripts\activate
   
   # macOS/Linux
   source venv/bin/activate
   ```

3. **Installer les dépendances**
   ```bash
   pip install -r requirements.txt
   ```

4. **Lancer l'application**
   ```bash
   streamlit run app.py
   ```

5. **Accéder à l'application**
   ```
   🌐 http://localhost:8501
   ```

### 🐳 **Installation avec Docker (Optionnel)**

```bash
# Construction de l'image
docker build -t gestionnaire-projets-ia .

# Lancement du conteneur
docker run -p 8501:8501 gestionnaire-projets-ia
```

## 📊 Architecture

### 🏗️ **Structure du Projet**
```
gestionnaire-projets-ia/
├── 📄 app.py                    # Application principale Streamlit
├── 📄 crm.py                    # Module CRM (contacts, entreprises, interactions)
├── 📄 style.css                 # Styles CSS personnalisés
├── 📄 requirements.txt          # Dépendances Python
├── 📄 Procfile                  # Configuration déploiement
├── 📄 runtime.txt               # Version Python
├── 📄 health_check.py           # Tests de santé application
├── 📄 .gitignore               # Fichiers ignorés par Git
├── 📄 README.md                # Documentation (ce fichier)
├── 📁 .streamlit/
│   └── 📄 config.toml          # Configuration Streamlit
├── 📁 data/                    # Données persistantes (créé automatiquement)
│   ├── 📄 projets_data.json    # Base de données projets
│   ├── 📄 crm_data.json        # Base de données CRM
│   └── 📄 inventaire_v2.json   # Base de données inventaire
└── 📁 docs/                    # Documentation supplémentaire
    ├── 📄 installation.md      # Guide installation détaillé
    ├── 📄 api.md               # Documentation API
    └── 📄 contributing.md      # Guide de contribution
```

### 🔄 **Flux de Données**
1. **Interface Utilisateur** (Streamlit) ↔️ **Logique Métier** (Python)
2. **Stockage Local** (JSON) ↔️ **Gestionnaires de Données** (Classes Python)
3. **Visualisations** (Plotly) ↔️ **Données Traitées** (Pandas)

### 🔐 **Sécurité**
- Validation des entrées utilisateur
- Sanitisation des données
- Gestion d'erreurs robuste
- Logs d'audit des actions

## 🌐 Déploiement

### 🚀 **Déploiement Automatique sur Render**

1. **Fork ce repository** sur votre compte GitHub
2. **Connectez-vous** à [Render.com](https://render.com)
3. **Créez un nouveau Web Service** :
   - Repository : Votre fork
   - Build Command : `pip install -r requirements.txt`
   - Start Command : `streamlit run app.py --server.port=$PORT --server.address=0.0.0.0`
4. **Déployez** et accédez à votre URL personnalisée

### ⚡ **Déploiement sur Heroku**

```bash
# Installation Heroku CLI
curl https://cli-assets.heroku.com/install.sh | sh

# Création application
heroku create votre-app-name

# Configuration buildpack
heroku buildpacks:set heroku/python

# Déploiement
git push heroku main

# Ouverture application
heroku open
```

### 🔧 **Variables d'Environnement**

Pour la production, configurez :
```bash
PORT=8501                    # Port application
STREAMLIT_SERVER_HEADLESS=true
STREAMLIT_SERVER_ENABLE_CORS=false
```
